\chapter{BDD-ТЕСТИРОВАНИЕ} \label{ch1}

В данной главе описано планирование и реализация BDD-тестирования
(Behaviour-Driven Development) приложения алгоритма Манакера.
Описывается методология BDD, синтаксис языка Gherkin, выбор инструментария,
а также разработанные тестовые сценарии для трёх ключевых функциональных
областей системы.


\section{Основы методологии BDD и язык Gherkin} \label{ch1:sec1}

Пусть дана строка $s = s_0 s_1 \dots s_{n-1}$ длины $n$.
Требуется определить:
\begin{enumerate}
    \item массив радиусов нечётных палиндромов с центрами в каждой позиции строки;
    \item массив радиусов чётных палиндромов с центрами между соседними символами;
    \item наибольшую палиндромную подстроку;
    \item количество палиндромных подстрок;
    \item дополнительные характеристики строки, полезные пользователю при анализе результата.
\end{enumerate}

Под палиндромной подстрокой понимается непрерывный фрагмент исходной строки, читающийся одинаково слева направо и справа налево.
Для нечётного палиндрома центр совпадает с символом строки, а для чётного располагается между двумя соседними символами.
Задача поиска палиндромных подстрок относится к задачам обработки строк и находит применение при анализе текстов, биологических последовательностей и повторяющихся симметричных шаблонов~\cite{gusfield1997}.

Наивный способ решения заключается в рассмотрении каждого возможного центра и последовательном расширении границ палиндрома.
Такой подход имеет асимптотику $O(n^2)$ в худшем случае.
Для длинных строк это приводит к избыточным затратам времени.
Алгоритм Манакера позволяет вычислить массивы радиусов всех палиндромов за линейное время $O(n)$, используя информацию о ранее найденных палиндромах~\cite{manacher1975}.

\section{Конкурирующие способы решения задачи} \label{ch1:sec2}

В соответствии с заданием на практическую работу приложение должно содержать не менее двух различных способов решения основной задачи.
Для проекта выбраны два алгоритма:
\begin{enumerate}
    \item алгоритм расширения от центра;
    \item алгоритм Манакера.
\end{enumerate}

Первый алгоритм используется как базовый эталон для сравнения результатов при тестировании.
Он последовательно рассматривает все возможные центры палиндромов и расширяет границы, пока символы совпадают.
В худшем случае его трудоёмкость составляет $O(n^2)$, однако он прост по структуре и удобен как проверочный алгоритм.

Второй алгоритм -- алгоритм Манакера -- является основным производственным решением.
Он выполняется отдельно для нечётных и чётных палиндромов, поддерживая текущий интервал $[l, r]$ с максимальной правой границей и зеркальную позицию относительно центра.
Благодаря этому алгоритм избегает повторной работы и обеспечивает линейную сложность.

\section{Алгоритм Манакера} \label{ch1:sec3}

Алгоритм Манакера выполняется отдельно для массивов $d_{odd}$ и $d_{even}$.
Для нечётного случая $d_{odd}[i]$ обозначает радиус палиндрома с центром в позиции $i$, включая центральный символ.
Для чётного случая $d_{even}[i]$ соответствует палиндрому с центром между символами $i-1$ и $i$.

Общая идея алгоритма включает следующие шаги:
\begin{enumerate}
    \item инициализировать границы $l$ и $r$ текущего палиндрома с максимальной правой границей;
    \item для каждой позиции определить стартовый радиус:
    если позиция находится вне интервала $[l, r]$, радиус задаётся начальным значением;
    если позиция попадает внутрь интервала, используется значение симметричной позиции;
    \item расширять палиндром, пока символы по обе стороны совпадают;
    \item при необходимости обновить текущие границы $l$ и $r$;
    \item после завершения обработки всех позиций использовать массивы радиусов для восстановления наибольшего палиндрома и подсчёта числа палиндромных подстрок.
\end{enumerate}

\section{Рекомендательная информация для пользователя} \label{ch1:sec7}

Согласно заданию приложение должно формировать полезную рекомендательную информацию.
В проекте предусмотрены следующие типы рекомендаций:
\begin{enumerate}
    \item если наибольший палиндром совпадает со всей строкой, выводится сообщение, что строка целиком симметрична;
    \item если длина наибольшего палиндрома составляет значительную долю строки, приложение сообщает о высокой палиндромной плотности;
    \item если в строке преобладают только короткие палиндромы длины 1--2, пользователь получает сообщение о слабой симметричности структуры;
    \item если длинный глобальный палиндром отсутствует, приложение сообщает о наличии нескольких локальных палиндромных областей.
\end{enumerate}

Наличие таких сообщений повышает прикладную ценность программы и делает результат полезнее, чем простой вывод массивов радиусов.

\section{Структура приложения} \label{ch1:sec8}

Проект реализуется на JavaScript в среде Node.js.
При проектировании выделены следующие модули:
\begin{itemize}
    \item \texttt{src/analyzers/manacherOdd.js} -- вычисление нечётных радиусов;
    \item \texttt{src/analyzers/manacherEven.js} -- вычисление чётных радиусов;
    \item \texttt{src/analyzers/centerExpansion.js} -- эталонный алгоритм расширения от центра;
    \item \texttt{src/services/palindromeService.js} -- агрегирование результатов;
    \item \texttt{src/io/jsonReader.js} -- загрузка входного JSON;
    \item \texttt{src/io/resultWriter.js} -- вывод отчёта в консоль и файл;
    \item \texttt{src/ui/consoleUi.js} -- консольное меню;
    \item \texttt{src/index.js} -- точка входа.
\end{itemize}

Такое разбиение позволяет независимо тестировать вычислительную логику, операции ввода-вывода и пользовательский интерфейс.

\section{Выводы} \label{ch1:conclusion}

В данной главе выполнено проектирование консольного приложения для анализа палиндромных подстрок.
Определены два конкурирующих способа решения задачи: расширение от центра и алгоритм Манакера.
Описаны варианты использования, структура меню, форматы входных и выходных JSON-данных, а также механизм формирования рекомендаций для пользователя.
Полученная архитектура удовлетворяет требованиям задания и служит основой для последующего проектирования тестов и реализации программы.
