\section{Алгоритмы генерации наборов данных} \label{ch2:sec2}

Для нагрузки выбраны строки четырёх типов, так как именно структура строки существенно влияет на алгоритм расширения от центра:
\begin{itemize}
    \item \texttt{uniform} --- строка из одинаковых символов; является худшим случаем для расширения от центра, так как почти каждый центр расширяется далеко;
    \item \texttt{alternate} --- чередование символов \texttt{ab}; создаёт большое число нечётных палиндромов;
    \item \texttt{mirror} --- строка с зеркальной структурой; содержит крупный палиндромный фрагмент;
    \item \texttt{random-like} --- псевдослучайный повтор алфавитного шаблона; содержит мало длинных палиндромов.
\end{itemize}

Генерация реализована в модуле \texttt{scripts/stringDataset.js}. Скрипт \texttt{scripts/generateDatasets.js} создаёт JSON-файлы в каталоге \texttt{datasets}. Каждый файл содержит имя набора, тип, длину и поле \texttt{input}.

\section{Предварительный эксперимент} \label{ch2:sec4}

Предварительный эксперимент нужен для выбора максимальных границ, при которых сравнение остаётся воспроизводимым и не превращается в ожидание десятков минут. Для алгоритма Манакера рост длины строки практически линейный, поэтому даже наборы в десятки тысяч символов обрабатываются быстро. Ограничивающим алгоритмом является расширение от центра.

План предварительного эксперимента:
\begin{enumerate}
    \item сгенерировать строки разных типов и размеров;
    \item выполнить оба алгоритма на одинаковых входах;
    \item проверить совпадение найденного самого длинного палиндрома;
    \item зафиксировать время выполнения и отношение времени эталонного алгоритма к времени Манакера;
    \item выбрать максимальный набор, на котором разница уже показательна, но эксперимент остаётся быстрым.
\end{enumerate}

В качестве максимального набора выбран \texttt{UNI\_16000\_A}. Он демонстрирует худший случай для расширения от центра: большое число центров приводит к квадратичному росту числа сравнений символов. При этом алгоритм Манакера сохраняет линейное поведение.

\section{Профилирование} \label{ch2:sec5}

Профилирование реализовано в \texttt{scripts/performanceCore.js}. Для каждого набора измеряются:
\begin{itemize}
    \item время расчёта алгоритмом Манакера;
    \item время расчёта алгоритмом расширения от центра;
    \item совпадение найденного палиндрома;
    \item приблизительный объём используемой памяти кучи Node.js;
    \item ускорение Манакера относительно эталонного алгоритма.
\end{itemize}

Запуск эксперимента выполняется командой:
\begin{center}
\texttt{npm run perf}
\end{center}

Эта команда формирует JSON- и CSV-отчёты в каталоге \texttt{reports/performance}.

\section{Выводы} \label{ch2:conclusion}

Для экспериментов подготовлены датасеты разных типов, влияющих на поведение алгоритмов. Набор \texttt{uniform} выбран как наиболее тяжёлый для расширения от центра, а \texttt{random-like} используется как контрольный случай с малым числом длинных палиндромов. План эксперимента позволяет одновременно проверить корректность, быстродействие и практическую применимость алгоритма Манакера.
