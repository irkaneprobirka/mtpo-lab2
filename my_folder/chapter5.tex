\section{Алгоритм Манакера} \label{ch5:sec2}

Алгоритм Манакера вычисляет радиусы всех палиндромов за линейное время. В проекте он разделён на две функции:
\begin{itemize}
    \item \texttt{manacherOdd(input)} возвращает массив радиусов нечётных палиндромов;
    \item \texttt{manacherEven(input)} возвращает массив радиусов чётных палиндромов.
\end{itemize}

Для каждой позиции поддерживается текущий палиндромный интервал $[l,r]$ с максимальной правой границей. Если позиция находится внутри этого интервала, стартовый радиус берётся из зеркальной позиции, после чего выполняется только необходимое дополнительное расширение. Именно повторное использование уже найденной информации обеспечивает сложность $O(n)$.

\section{Эталонный алгоритм расширения от центра} \label{ch5:sec3}

Алгоритм расширения от центра перебирает каждый возможный центр палиндрома и двигает левую и правую границы, пока символы совпадают. Для одиночных или псевдослучайных строк он может работать быстро, но в худшем случае, например для строки из одинаковых символов, выполняет квадратичное число сравнений.

В данной работе этот алгоритм используется как:
\begin{enumerate}
    \item эталон корректности для сравнения результата Манакера;
    \item контрастный алгоритм для нагрузочного эксперимента;
    \item источник BDD-сценария, в котором два решения наукоемкой задачи сравниваются на одинаковых данных.
\end{enumerate}

\section{Формат входных и выходных данных} \label{ch5:sec4}

Входной файл имеет JSON-формат:
\begin{lstlisting}[language=JavaScript]
{
  "input": "abacaba"
}
\end{lstlisting}

Результат содержит исходную строку, массивы радиусов, самый длинный палиндром, количество палиндромных подстрок и рекомендацию:
\begin{lstlisting}[language=JavaScript]
{
  "algorithm": "manacher",
  "input": "abacaba",
  "oddRadii": [1, 2, 1, 4, 1, 2, 1],
  "evenRadii": [0, 0, 0, 0, 0, 0, 0],
  "longestPalindrome": {
    "value": "abacaba",
    "start": 0,
    "end": 6,
    "length": 7
  },
  "palindromeCount": 12
}
\end{lstlisting}

\section{Выводы} \label{ch5:conclusion}

Приложение содержит два алгоритма решения одной задачи, модульный ввод-вывод, консольный интерфейс, unit-тесты и BDD-сценарии. Такая структура позволяет проверять корректность на нескольких уровнях: от массивов радиусов до пользовательских историй и нагрузочного сравнения.
